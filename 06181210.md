## §1 Phase 1 — 最小可用生成闭环（MVP 内核）

**目标**：登录用户在简易模式经 `apps/api` + Worker 调 MiniMax 生成一首歌/一段语音，看到实时进度，试听，存入资产库；**失败必退额度、成功才扣、重复提交幂等**。把生成从 `apps/web` 的同步路由迁到异步内核。

### 1.1 基础设施接通（health 转真实探针）
- `apps/api`：新增 `PrismaModule`/`PrismaService`（onModuleInit 连接、$queryRaw ping）。
- 新增 `RedisModule`（ioredis 单例）、`StorageModule`（S3/MinIO 客户端，建桶）。
- `health` 升级为真实 ping：DB / Redis / S3 / （可选）MiniMax 轻探针。
- 依赖新增（仅声明，落地时安装）：`@prisma/client` 已装；`ioredis`、`bullmq`、`@aws-sdk/client-s3`、`@aws-sdk/s3-request-presigner`、`argon2`(或 bcrypt)、`@nestjs/jwt`、`@nestjs/websockets`+`@nestjs/platform-socket.io`、`cookie-parser`、`zod`。

### 1.2 auth（最小集） · `apps/api/src/modules/auth/**`
- 用例：`register / login / refresh / logout`；access JWT（短）+ refresh（httpOnly cookie，可轮换）。
- 守卫：`JwtAuthGuard`、`@CurrentUser()` 装饰器、`RolesGuard`（雏形）。
- 契约：复用 `@music/contracts` 的 `registerInput`/`loginInput`/`AuthResult`/`PublicUser`。
- REST：`POST /auth/register`、`POST /auth/login`、`POST /auth/refresh`、`POST /auth/logout`。
- 验收：注册/登录/会话保持；密码 argon2 哈希；错误码 `AUTH_EMAIL_TAKEN`/`AUTH_INVALID_CREDENTIALS`。

### 1.3 entitlement（额度账本 · 三段式） · `apps/api/src/modules/entitlement/**`
- 数据源：`CreditLedger`（append-only，唯一可信余额）+ `credit_balances` Redis 镜像。
- 服务：`getBalance(userId)`、`hold(userId, cost, ref)`、`commit(holdId)`、`refund(holdId)`、`grantMonthly(userId, plan)`、`consume`。
- 成本表（配置化，入 `Plan.features` 或独立配置）：音乐 = N credits；TTS 每 ≤K 字符 = M credits。
- 内置 `free` 套餐种子（注册送月度额度）。
- 不变量（P3 单测覆盖）：余额 = Σgrant − Σcommit；hold 未结算不可重复 commit；失败必 refund。
- 验收：并发 hold 同一用户不超发；额度为 0 拒绝并返回 `ENTITLEMENT_INSUFFICIENT_CREDITS{balance,required}`。

### 1.4 provider（MiniMax 领域封装） · `apps/api/src/modules/provider/**`
- 注入 `MiniMaxClient`（apiKey 取自 `loadEnv()`，绝不下发前端）。
- 领域参数 ↔ MiniMax 请求映射（music / tts）；`onUsage` → `telemetry`/`api_usage` 成本记账。
- 错误归类已在 sdk：`transient`→重试/排队；`permanent`→立即退款。

### 1.5 media · `apps/api/src/modules/media/**`
- S3 上传、可过期签名播放/下载 URL；远端音频下载转存（沿用“无对象存储则直用 MiniMax URL，有则转存”策略）。
- 波形 peaks（服务端 ffmpeg / 音频库，存 `Asset.waveform` JSON）。

### 1.6 generation 状态机 · `apps/api/src/modules/generation/**`
- `POST /generation/jobs`（Header `Idempotency-Key`）：zod 校验 → 解析最简预设 → 估算 cost → `entitlement.hold` → 建 `GenerationJob(queued)` → 入 BullMQ → 返回 `{jobId,status,creditCost}`。幂等命中返回既有 job。
- `GET /generation/jobs/:id` → `JobView`。
- 状态机：`queued→processing→(succeeded|failed|canceled)`；非法迁移抛 `GENERATION_INVALID_STATE`。

### 1.7 Worker 处理器 · `apps/worker/src/processors/generation.processor.ts`
- `processing` → provider 调用 → 成功：下载/落存储 + 算时长/波形 → 建 `Asset(draft)` → `succeeded` → `entitlement.commit` → 发 `job.updated`。
- 失败：`failed` + `entitlement.refund` + 发事件 + 按 transient 重试（指数退避，上限）。

### 1.8 实时进度 · `apps/api/src/modules/realtime/**`
- Socket.IO 网关，房间 `user:{userId}`，事件 `job.updated`（`JobUpdatedEvent`）。
- **轮询回退**：断线不影响最终一致（`GET /generation/jobs/:id`）。

### 1.9 assets · `apps/api/src/modules/assets/**`
- 列表/详情/删除草稿；试听签名 URL。

### 1.10 web 简易模式 v1（接 API） · `apps/web`
- 新增 API 客户端（TanStack Query）+ 鉴权会话；把现有 `/studio` 表单改为：提交 job → 订阅 WS 进度卡片 → 试听播放器 → “我的草稿”。
- 现有 `apps/web/src/app/api/*` 同步生成路由标记为 legacy（保留可回滚，逐步下线）。

**Phase 1 验收**（母计划 §7/§11）：免费用户成功生成音乐与语音；额度扣减/失败退款正确；可试听；幂等生效；进度实时 + 轮询回退可用。

## §2 Phase 2 — 完善为完整产品（按子里程碑）

### 2A 发布与消费端基础
- `catalog/tracks`：`POST /tracks/publish`（draft asset → `Track(pending_review)`）；一键发布对话框（标题/封面/歌词/曲风/标签/语言/可见性，AI 标识强制 true）。
- `moderation`（先审后发，可配）：发布触发自动预筛（歌词文本策略 + 封面图扫描）→ 命中进 `ModerationCase` 人工队列；通过 → published。
- 消费端 v1（`apps/web`）：曲库/作品详情、**全局持久播放器**（Root Layout 单例 `<audio>` + Zustand `usePlayerStore`，路由切换不断播、队列/进度/顺序-随机-单曲）、创作者主页、基础搜索（PG FTS：标题/创作者/标签）。
- `social` v1：点赞、收藏到歌单、关注；`play` 事件上报（去重 + 最小播放时长阈值，反哺榜单）。

### 2B 商业化闭环
- `billing/subscription` + `payments`：`plans(free/monthly/yearly)`、下单、支付适配（先接一个真实渠道 + 适配位）、Webhook 幂等（`WebhookEvent` 去重）、订阅生命周期（trial/active/past_due/canceled/expired）。
- `entitlement` 完整化：付费月度额度授予 + 周期重置（cron）+ 权益门（专业模式/克隆/并发/最大时长/商用/免水印/优先队列）。
- web：定价页、订阅管理、额度余额与用量、不足引导升级。

### 2C 专业模式与预设
- 创作端专业模式：歌词结构编辑器（`[verse]/[chorus]/[bridge]`）、参考音频/Cover（`music-cover`）、曲风/情绪/时长/采样率/比特率/纯音乐、TTS 高级项（语速/音量/音调/情绪/发音词典/停顿 `<#x#>`/语言增强）、项目与版本管理（多版本/重生成/A-B 对比）。
- `presets`：官方预设（运营维护）+ 用户自存；简易/专业一键套用。
- `voices`：系统音色目录（缓存 MiniMax 音色清单 + 试听 + 标签/语言/性别筛选）。

### 2D 榜单与社区
- `charts`：热/飙升/新歌/分类/创作者；Redis sorted set 实时热度 + 定时按时间衰减算分写 `ChartEntry`；防刷（去重/最小时长/频率上限/异常流量）。
- 分类聚合页；`comments`（层级/审核态/举报）。

### 2E 运营后台 · `apps/admin`（AntD / Refine）
- 用户/订阅/额度/内容审核队列/曲库与榜单运营/预设与音色目录/数据看板/MiniMax 成本看板/系统配置（feature flag 热生效）/审计日志。受 admin RBAC 保护。

**Phase 2 验收**：可发布、可聆听、可付费、可看榜、专业模式与预设可用、后台覆盖全部内容。

## §5 跨阶段约定（贯穿执行）

- **错误码**：一律走 `@music/contracts` 的 `ApiErrorBody{code,message,details?,traceId}`；前端按 `code` 驱动 UI，不依赖 `message`。
- **额度**：append-only `CreditLedger` 为唯一真相；任何扣/退留痕；`hold→commit/refund` 失败必退。
- **幂等**：生成用 `Idempotency-Key`；支付用 `WebhookEvent.eventId` 唯一。
- **迁移**：向前兼容（先加后用；删除两步：停用→下版移除）；不手改历史迁移。
- **安全**：第三方密钥仅后端（`loadEnv()`），绝不进 `NEXT_PUBLIC_*`；音频走可过期签名 URL。
- **降级**：MiniMax 限流/故障→排队 + 提示，不崩；provider 抽象预留备用供应商接口（v1 不实现）。
- **功能开关**：新功能用 feature flag 包裹，问题时关开关即回滚体验。
