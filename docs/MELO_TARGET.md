# Melo target site map

Melo is a Chinese AI music product inspired by the breadth of Suno's public product experience. The goal is an original Chinese implementation with equivalent user journeys, not a pixel copy or reuse of protected assets.

## Primary navigation

- Discover: editorial picks, trending songs, genre channels, playlists, creator activity, creation challenges.
- Create: prompt-based song generation, custom lyrics, instrumental mode, voice/persona selection, upload/reference audio, generation queue, drafts, versions.
- Library: user's songs, liked songs, playlists, drafts, uploads, deleted items, bulk actions.
- Search: songs, creators, playlists, genres, and prompt ideas.
- Charts: hot, new, creator, genre, and challenge charts.
- Pricing: plan comparison, credits, commercial usage, feature gates.
- Account: profile, subscription, credits, settings, safety and consent.

## Detail pages

- Track: playable song detail, lyrics, creator, style tags, actions for like/share/remix/extend/download, comments and related tracks.
- Playlist: playlist header, track table, owner metadata, play all and save actions.
- Creator: profile, featured tracks, playlists, stats, style identity.
- Category: genre landing page with chart, playlists, and prompt starters.

## Create workspace

- Simple mode: prompt, genre, mood, vocal, lyrics mode, duration, output format.
- Custom mode: structured lyrics editor, section controls, instrumental toggle, advanced audio settings.
- Voice/persona: saved voices, style identity, reference constraints.
- Upload/remix tools: upload audio, cover/remix/extend/remaster actions represented in UI.
- Drafts/projects: generation history, versions, compare, continue from previous job.

## Current implementation priorities

1. Replace broken Chinese copy and old brand references with Melo.
2. Make Discover, Search, Charts, Library, Create, Pricing, Account, Track, Playlist, Creator, and Category coherent.
3. Add compatibility routes for target-like URLs where useful, such as `/create` and `/explore`.
4. Verify local rendering with the seeded fallback catalog when no backend is running.
