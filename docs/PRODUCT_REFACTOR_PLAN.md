# AI 原生音乐平台重构计划

## 产品定位

本项目重构为三端一体的音乐平台：

- A：音乐消费平台，服务歌迷、音乐爱好者、音频使用者。
- B：AI 音乐与音声创作平台，服务创作者、音乐人、短视频/播客/游戏/影视音频生产者。
- C：运营后台，服务内容审核、榜单运营、用户/订阅/额度/成本管理。

当前阶段只实现 Web 端完整闭环；Android/iOS 暂保留接口契约和数据模型，不开发客户端。

## A：音乐消费平台

Web 必须像常规音乐平台，而不是 AI 工具页。

核心模块：

- 首页推荐：热门榜、新歌、分类入口、AI 原创专区。
- 榜单：热歌榜、新歌榜、飙升榜、分类榜、AI 原创榜。
- 分类：流行、国风、R&B、电子、Lo-fi、纯音乐、影视/游戏配乐等。
- 歌曲详情：播放、歌词、AI 标识、作者、标签、收藏、评论、分享。
- 创作者主页：作品、粉丝、播放量、简介。
- 歌单：平台歌单、用户歌单、收藏歌单。
- 搜索：歌曲、创作者、标签、风格。
- 全局播放器：跨路由持续播放、队列、进度、音量、播放模式。

## B：AI 创作平台

简易模式：

- 一句话需求
- 风格、情绪、人声、时长
- 纯音乐/歌词模式
- AI 写词
- 一键生成
- 试听、重新生成、保存草稿、一键发布

专业模式：

- 歌词结构编辑
- BPM、Key、采样率、码率、格式、最长时长
- 参考音频、风格参考、Cover 模式
- TTS 音声、语速、音量、音调、情绪、发音词典、停顿控制
- 项目、多版本、A/B 对比
- 官方预设与用户自定义预设

## C：运营后台

核心模块：

- 用户、创作者、角色权限
- 作品、草稿、发布审核
- 评论、举报、风控
- 榜单、分类、推荐位
- 预设、音色
- 订阅、订单、额度
- MiniMax 成本、失败任务、API 调用监控
- 审计日志、Feature Flag

## 关键闭环

创作闭环：

```txt
登录 -> 免费额度 -> 提交生成 -> hold 额度 -> Worker 调 MiniMax
-> 保存 Asset -> commit 额度 -> 草稿试听 -> 发布审核 -> 进入音乐平台
```

失败闭环：

```txt
提交生成 -> hold 额度 -> 失败/超时/拒绝 -> failed -> refund -> 用户重试 -> 后台可追踪
```

```txt
/pricing              定价

```
